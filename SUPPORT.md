# Support

## Getting Help

If you have questions about Open Operational State, the specification, or related tooling, here are the best ways to get help.

### GitHub Discussions

For general questions, ideas, and community conversation, use GitHub Discussions on the relevant repository.

### Issues

For bug reports and feature requests, use the issue templates provided in each repository.

### Email

For matters that do not fit into public channels:

**support@open-operational-state.org**

### Security Issues

For security vulnerabilities, see [SECURITY.md](SECURITY.md). Do not open public issues for security reports.

## Repository Guide

| Repository | Purpose |
|---|---|
| [governance](https://github.com/open-operational-state/governance) | Charter, governance model, terminology, roadmap |
| [status-spec](https://github.com/open-operational-state/status-spec) | Technical specification for operational-state standard |
| [status-conformance](https://github.com/open-operational-state/status-conformance) | Conformance definitions, fixtures, test taxonomy |
| [status-tooling](https://github.com/open-operational-state/status-tooling) | Vendor-neutral reference tooling (parsers, validators, etc.) |
